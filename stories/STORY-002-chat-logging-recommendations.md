# Story: Chat Logging and KB Recommendations

**Story ID:** STORY-002
**Date:** 2026-05-12
**Status:** In Progress

---

## User story

**As an** admin
**I want** to see what users have been asking the chatbot, and receive AI-generated suggestions for new knowledge base entries based on those conversations
**So that** I can identify gaps in coverage and improve the knowledge base over time without manually reviewing every log

---

## Acceptance criteria

- [ ] Every user message and the bot's response are logged to the database with site_id and timestamp
- [ ] Admin portal shows a "History" tab per site listing recent conversations in reverse chronological order
- [ ] Admin portal shows a "Recommendations" tab with AI-suggested KB entries based on chat history
- [ ] Recommendations show category, question, and a draft answer
- [ ] Each recommendation has an "Add to KB" button that pre-fills the entry modal

---

## Test specifications

### Backend tests
- [ ] `GET /api/logs/:siteId` returns `200` with an array
- [ ] `GET /api/logs/unknown` returns `200` with empty array
- [ ] `GET /api/recommendations/:siteId` returns `200 { recommendations: [], message: ... }` when no logs exist
- [ ] `GET /api/recommendations/:siteId` returns `500` when ANTHROPIC_API_KEY is not set and logs exist

---

## Out of scope

- GDPR / data retention policy (demo only)
- Per-session grouping of conversation history
- Export of chat logs to CSV
- Real-time log streaming

---

## Notes / context

Recommendations are generated by calling Claude with the recent chat logs and existing KB as context. Claude returns a JSON array of suggested entries. The endpoint is non-streaming (regular request/response). When no chat history exists the endpoint returns an empty recommendations array without calling Claude.

Logging is non-fatal — if the INSERT into chat_logs fails it is caught and logged to console but does not affect the chat response.
